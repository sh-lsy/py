# Python 课程助手 (Python Course Helper)

此技能用于协助管理和扩展 Python 学习项目，确保为初学者（小白）提供清晰、直观的学习体验。

## 1. 项目结构 (Project Structure)

项目分为以下几个模块（目录）：

- 📂 **`01_base`** (基础 Python): 变量、循环、函数等核心概念。
- 📂 **`02_advanced`** (进阶 Python): 面向对象、装饰器、多线程等。
- 📂 **`03_data`** (数据分析): NumPy, Pandas, Matplotlib。
- 📂 **`04_crawler`** (网络爬虫): Requests, BeautifulSoup, Selenium。
- 📂 **`05_practical`** (实战项目): 综合性的小工具或应用。

## 2. 内容生成规范 (Content Guidelines) - **核心要求**

为了让小白用户也能轻松看懂，生成内容时**必须**遵守以下规范：

### 2.1 文档编写 (Documentation)

在更新 `README.md` 时：

- **多用列表**: 步骤、要点尽量使用无序列表 (`-`) 或有序列表 (`1.`)。
- **多用表格**: 对比概念（如 List vs Tuple）或列举参数时，**必须**使用 Markdown 表格。
- **适当代码块**: 关键语法、参数说明或示例代码，**必须**使用 Markdown 代码块包裹，以提高可读性。
- **通俗易懂**: 避免堆砌专业术语，使用生活中的类比（如“变量是贴标签”）。

### 2.2 可视化辅助 (Visual Aids)

- **强制配图**: 对于抽象、复杂的概念（如内存引用、切片索引、函数调用栈、数据流向），**必须**提供图片辅助理解。
- **生成方式**:
  1. 创建一个 `generate_diagrams.py` (如果该目录还没有)。
  2. 使用 `matplotlib` 或 `graphviz` 编写代码生成 PNG 图片。
  3. 保存到 `assets/` 目录。
  4. 在 `README.md` 中引用图片 `![描述](assets/image.png)`。

### 2.3 代码质量 (Code Quality)

- **注释丰富**: 每行关键代码都要有注释。
- **分步执行**: 复杂的逻辑拆分成小步骤演示。
- **覆盖全面**: 必须包含所有的核心知识点、常见误区（如 `return` 后不执行）、边界情况（如无返回值时返回 `None`）。

## 3. 工作流 (Workflow)

当用户说“我想学习 [主题]”时：

1.  **定位**: 确定属于哪个模块（`01_base`/`02_advanced` 等）。
2.  **检查**: 查看当前目录下的文件序号（如 `11_loops.py`），新文件顺延为 `12_[topic].py`。
3.  **编写代码**: 创建 Python 学习脚本，包含详细注释。
4.  **生成图解**: (如果概念抽象) 编写/更新绘图脚本并生成图片。
5.  **更新文档**: 在 `README.md` 中添加新章节，包含：
    - 知识点列表
    - 概念对比表
    - 刚刚生成的图片
    - 代码文件链接

## 4. 示例

如果用户学习“列表切片”：

- **代码**: `05_lists.py` 展示 `list[1:3]`。
- **图解**: 生成一张图片展示索引 `0, 1, 2` 对应的元素位置。
- **文档**: 用表格列出 `start`, `stop`, `step` 的含义。
